I Built a Live Data Pipeline From Scratch — Here's How Spectral Started

Craig Kelley · March 20, 2026

I spent 23 years in underground utilities and infrastructure contracting. Field work, long days, physical wear — that was my world.

My body has limits. I knew that before the job did. So I started teaching myself to code to build a way out before I hit them.

The first thing I shipped that wasn’t a toy was a cold email system in Python: SMTP, personalization tokens, deduplication so the same lead didn’t get hammered, basic guardrails. It worked. Someone paid me for it.

That unlocked something. I didn’t want another one-off script — I wanted a real pipeline: scrape sources, enrich what mattered, verify it, clean it, and hand back structured data someone could use.

I called it Spectral. Config-driven, modular, built as a monorepo: a scraper, an enrichment layer, a verification step, and a CSV cleaner that could take messy input and make it consistent.

The goal was always two-fold: run it as a service engine for client jobs, and eventually sell packaged lead lists as a data product. Same core, different packaging.

There were dead ends — approaches that looked right until scale or edge cases broke them. I won’t pretend it was a straight line. But the stack settled: a FastAPI backend, deployed on a VPS, with the pipeline wired so jobs could move through it end to end.

It’s live now at spectral.crkdev.com. Upload a file or point it at a URL, run the flow, download results. That’s the product in one sentence.

This post is the first in a series where I document what I build next — the features, the tradeoffs, and what actually ships.
